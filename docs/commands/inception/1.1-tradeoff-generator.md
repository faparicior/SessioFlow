# üé≠ The Roles to Simulate
Since the detailed personas and empathy maps do not exist yet, the roles must deduce their arguments strictly from the high-level Vision document:

1.  **The Agile Coach (Facilitator)**
    *   *Goal:* Enforce the "Golden Rule" (Only one checkmark per column 1-N).
    *   *Behavior:* Forces the team to prioritize. If everything is important, nothing is.

2.  **The Product Owner (Business)**
    *   *Goal:* Champion the "Goals" and "Values" in the Vision.
    *   *Behavior:* If the Vision says "Low Cost" or "MVP", they must aggressively down-rank expensive attributes like Scalability or infinite Flexibility.

3.  **The User Advocate (UX/Design)**
    *   *Goal:* Represent the "Who" defined in the Elevator Pitch.
    *   *Behavior:* Interpret the user's needs from the high-level description.
        *   *Example:* If the "Who" is "Busy non-technical managers", they fight for **Usability** and **Simplicity**.
        *   *Example:* If the "Who" is "Sysadmins", they might fight for **Flexibility**.

4.  **The Tech Lead (Engineering)**
    *   *Goal:* Assess the technical reality of the "Is/Is Not" constraints.
    *   *Transformation:* They usually start wanting "Performance/Security", but must concede if the "Product IS NOT" section explicitly rules out complex architecture (e.g., "Product IS NOT a complex enterprise system").

# ‚ö° The Process
1.  **Analyze Context:** Internalize the `1-product-vision-and-boundaries.md` file. Identify the core conflicts (e.g., "We want it cheap" vs "We want it secure").
2.  **Simulate Debate:** Have the roles debate the standard qualities: Cost, Usability, Simplicity, Security, Flexibility, Performance, Scalability.
3.  **Resolve:** Create a strict 1-to-7 ranking (1 = Most Important).
4.  **Validate & Self-Correct:** Before finalizing, audit your own result using the criteria from `docs/commands/inception/2-tradeoff-validator.md`.
    *   *Check:* Does each column have exactly one mark? (The Golden Rule)
    *   *Check:* Are specific trade-offs (e.g., Cost vs Scalability) aligned with the Product Vision's constraints?
    *   *Action:* If the validation reveals any "Rule Violations" or "Strategic Misalignments", you **must** revise the ranking until it is fully compliant.
