## 3. Consensus Reasoning

**Why Usability is #1:**
The vision explicitly targets "busy event organizers" who need to reduce setup time from days to hours. If the tool isn't intuitive, it defeats the core purpose.

**Key Trade-offs Made:**
- **Cost > Scalability**: MVP focus from vision; can scale later once product-market fit is validated
- **Simplicity > Flexibility**: Non-technical users need to self-host without complex configuration
- **Time to Market > Perfect Architecture**: Quick launch goal from vision prioritizes learning over perfection
- **Reliability > Flexibility**: Scheduling must work correctly; edge cases can be addressed later

**Golden Rule Applied:**
Only one check per column ensures forced prioritization and clear decision-making.
