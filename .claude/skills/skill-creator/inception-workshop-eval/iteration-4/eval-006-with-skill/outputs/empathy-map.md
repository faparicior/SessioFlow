### Profile
- **Age**: 72
- **Living Situation**: Lives alone in suburban home
- **Tech Experience**: Limited, uses basic smartphone
- **Physical Considerations**: Mild arthritis, declining vision

- **Overwhelmed** by too many features and options
- **Lonely** when technology creates barriers instead of helping
- **Proud** when she figures something out on her own
- **Discouraged** when progress is slow

## Key Pain Points for Elderly Users

### Physical Barriers
1. **Small touch targets** - Difficult for arthritic fingers
2. **Low contrast displays** - Hard to see with declining vision
3. **Fast voice responses** - Can't keep up with rapid speech
4. **Complex gestures** - Swipe, pinch, multi-tap are challenging

### Cognitive Barriers
1. **Too many options** - Decision paralysis
2. **Unclear icons** - Don't understand symbols
3. **Technical jargon** - Confusing terminology
4. **No clear undo** - Fear of making irreversible mistakes

### Emotional Barriers
1. **Fear of breaking** - Hesitant to try new features
2. **Embarrassment** - Don't want to ask for help
3. **Frustration** - Technology should be easier
4. **Isolation** - Feels left behind by technology

---

## Opportunity Areas for HomeHub

### Accessibility Improvements
1. **Voice-first interface** - Minimal screen interaction required
2. **Slower, clearer voice responses** - Adjustable speech rate
3. **Large, high-contrast buttons** - Physical or on-screen
4. **Haptic feedback** - Confirm actions through touch

### Simplicity Features
1. **Basic mode** - Show only essential functions
2. **One-touch actions** - "Good morning", "Good night" routines
3. **Visual guides** - Simple illustrations, not text-heavy
4. **Progressive disclosure** - Reveal features gradually

### Support Systems
1. **Family connection** - Remote assistance from relatives
2. **Simplified troubleshooting** - Step-by-step visual guides
3. **Human support option** - Easy access to live help
4. **Practice mode** - Safe environment to learn

### Emotional Considerations
1. **Patience** - No timeouts or error messages that feel judgmental
2. **Encouragement** - Positive reinforcement for learning
3. **Respect** - Treat users as capable, not deficient
4. **Independence** - Enable self-sufficiency, not dependence
